## Outcome

From the **Projects** screen the user selects an image asset, clicks **Use with AI**,
lands in **AI Workspace â†’ Image Upscaler** with the asset preloaded, runs the upscale,

## Provider chain

`jobs.processor.processAIJob()` resolves an ordered chain for `IMAGE_UPSCALE`:

1. `AITool.provider` if set (e.g. `dispatcher`)
2. `params.providerPreference` (per-job override; from the workbench)
3. `AI_PROVIDER_PRIORITY` env (default `dispatcher,comfyui,mock`)
4. `mock` is appended as a safety net

A provider whose `isConfigured()` returns false is skipped silently. A configured
provider that returns `{notConfigured:true}` (e.g. `DISPATCHER_NOT_CONFIGURED`) falls
through to the next link. A genuine runtime failure surfaces immediately and is
written to `AIJob.errorMsg`; the source asset's `status` is restored or set to `FAILED`,
and no empty version is created.

## Mock test (no dispatcher / no ComfyUI)

1. Leave `DISPATCHER_BASE_URL` and `COMFYUI_BASE_URL` unset.
2. Start backend: `npm run dev`.
3. In Projects, pick an image, click **Use with AI**.
4. Click **Upscale** in the workbench.
5. Job completes via `MockProvider`. The mock re-copies the source URL as the
   "upscaled" result so a real `AssetVersion` is created. `result.mockFallback === true`.

## Dispatcher test

1. Set `DISPATCHER_BASE_URL`, optionally `DISPATCHER_API_KEY`, and
   `COMFYUI_UPSCALE_WORKFLOW_JSON` (or `_PATH`).
2. POST `/api/assets/use-with-ai` (see smoke script).
3. The job moves through `queued â†’ starting_pod â†’ waiting_comfyui â†’ running â†’ done`.
4. Backend copies `result_url` (or `output_url` / `fileUrl` / `url`) into primary
   storage and creates a new `AssetVersion`.
