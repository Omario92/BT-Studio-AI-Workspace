# V0.4 Integration & Lifecycle Smoke Test

This document outlines the step-by-step procedure to run programmatic end-to-end smoke testing for BT Studio AI Workspace V0.4.

---

## 1. Overview
The lifecycle smoke test (`backend/scratch/assets.smoke.ts`) programmatically invokes the Fastify REST endpoints to verify:
* **Authentication**: Token generation via `POST /api/auth/login`.
* **Projects**: Active project retrieval via `GET /api/projects`.
* **Asset Upload Flow**: Production model creation using `POST /api/assets/upload` (simulating file key linking to S3/Local object storage).
* **Versioning**: Incrementing `AssetVersion` records via `POST /api/assets/:id/versions`.
* **Flow States**: Triggering reviews and verifying the required constraints (e.g., rejecting decisions when comments are missing).
* **Reviews and Comments Direct Fetching**: Confirming direct comment aggregation via custom relational endpoints.

## 2. Prerequisites & Setup

### A. Environment Configuration
Ensure your local backend `.env` matches the current Railway database proxy. The script points by default to `http://localhost:3001/api`.

Make sure your server is running with the proxy URL. Example:

Ensure the database has been seeded with seed accounts (e.g., `alice@btstudio.ai` and `password123` default credentials):

```bash
cd backend
npx tsx scratch/assets.smoke.ts
```

## 4. Manual Verification Steps

For graphical end-to-end verification, you can interact directly with the frontend:
1. Boot both backend and frontend environments locally:
   * **Backend**: `npm run dev` (running on `localhost:3001`).
   * **Frontend**: `npm run dev` in `front-end/` (running on `localhost:5173`).
2. Log in using `alice@btstudio.ai` / `password123`.
3. Open a Project and upload an image file using the direct drag-and-drop picker. You should see real-time progress percentages.
4. Go to **AI Workspace** and submit an Image Upscaling or Editing request. The request will automatically route through your active provider (Mock or RunPod/ComfyUI depending on your configuration) and display real-time background status polls until final storage completes.
